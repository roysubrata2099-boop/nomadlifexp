# 🔐 Security Policy

## Supported Versions

We actively provide security updates for the following versions of this project:

| Version | Supported |
|---------|----------|
| 5.1.x   | ✅ Yes   |
| 5.0.x   | ❌ No    |
| 4.0.x   | ✅ Yes   |
| < 4.0   | ❌ No    |

---

## 🚨 Reporting a Vulnerability

We take security issues seriously and appreciate responsible disclosure.

### 📩 How to report
If you discover a security vulnerability, please report it privately by:
- Opening a **private security advisory** on GitHub (preferred), or
- Contacting the maintainer directly (add your email here if available)

### 📌 What to include
To help us understand and fix the issue quickly, please include:
- Clear description of the vulnerability
- Steps to reproduce the issue
- Affected version(s)
- Possible impact
- Suggested fix (if any)

---

### ⏱ Response timeline
- Initial response: **within 48–72 hours**
- Status update: provided after investigation begins
- Fix timeline: depends on severity, but critical issues are prioritized immediately

---

## 🛠 Handling of reports

- If the vulnerability is **confirmed**, we will:
  - Work on a fix
  - Release a patch if needed
  - Credit the reporter (if desired)

- If the report is **not valid**, we will explain why and close it.

---

## 🙏 Thank you

We appreciate your effort in helping keep this project secure.
